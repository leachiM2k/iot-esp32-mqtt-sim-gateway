GPS wird asynchron auf dem Board verarbeitet: das gps-Kommando fordert nur einen Fix an, das Board akquiriert ihn im Hintergrund und publiziert /gps, sobald er steht (Kaltstart bis ~90 s). GNSS bleibt nach einer Abfrage an (für schnellere Folge-Fixes); gpsoff schaltet es aus, um Strom zu sparen.
ℹ️ Die Höhe wird über das EGM96-Geoidmodell von ellipsoidisch (WGS84) in orthometrische Höhe über NN umgerechnet. Der Geoid-Wert (Abweichung des Meeresspiegels vom Ellipsoid) wird in Klammern angezeigt.
Über diese Konsole ESP32 A7670 · MQTT Bridge

Lokale Web-Konsole für das ESP32-A7670-SMS/Anruf-Gateway. Die Kommunikation läuft über eine lokale Deno-Bridge, die MQTT-Credentials serverseitig hält und nur eine feste Kommando-Whitelist durchlässt. Boards werden automatisch erkannt (retained Messages).

Verfügbar (in Firmware integriert)

  • ✅ Gerätestatus anzeigen — Topic /info
  • ✅ Anrufstatus anzeigen — Topic /callstatus
  • ✅ Empfangene SMS auflisten — Topic /sms
  • ✅ Eingehenden Anruf annehmen — Kommando accept
  • ✅ Anruf ablehnen / auflegen — Kommando hangup
  • ✅ SMS senden — Kommando sms
  • ➕ Ausgehenden Anruf starten — Kommando call
  • ➕ Board neustarten — Kommando reboot
  • ➕ Eingehende Anrufernummer — Topic /checkresult
  • ➕ GPS-Position abfragen — Kommando gps → Topic /gps
  • ➕ GNSS ausschalten — Kommando gpsoff
  • ➕ VoLTE ein/aus — Kommando volte (enable) → Topic /volte

Nicht integriert / Einschränkungen

  • ❌ USSD senden — die Firmware hat sendUSSD(), ist aber nicht als MQTT-Kommando verdrahtet.
  • ℹ️ „Ablehnen" und „Auflegen" sind dasselbe Kommando (hangup / AT+CHUP).
  • ℹ️ Bridge nur auf 127.0.0.1 — nicht aus dem Netz erreichbar.
  • ℹ️ Publish-Rechte auf esp32/commands/<MAC> nötig.
  • ℹ️ Ausgehende SMS als GSM-7 — Umlaute/Emoji evtl. verstümmelt (Empfang korrekt via UCS2).
